Mayo 2: • Actividad: Se investigaron las implicaciones legales del reconocimiento facial. • Resultados: El proyecto cambió de enfoque debido a preocupaciones legales sobre privacidad, pasando de reconocimiento facial a un sistema auditivo para discapacitados visuales. • Problemas: Riesgos legales y de privacidad identificados. • Soluciones: Modificación del proyecto, descartando el reconocimiento facial. Junio 14: • Actividad: Decisión de usar la cámara ESP32 para el reconocimiento de imágenes y un sistema de respuesta auditiva. • Resultados: El sistema ayudaría a los usuarios a desplazarse con seguridad. • Problemas: Complejidad en crear una base de datos para la ESP32. • Próximos pasos: Investigar la ESP32 y evaluar alternativas si es necesario. Junio 20: • Actividad: Presentación del proyecto sin mentor. Se detallaron los componentes clave (cámara ESP32 y sistema auditivo). • Próximos pasos: Investigar más a fondo la ESP32 y realizar pruebas. Junio 27: • Actividad: Reunión con mentor para evaluar la viabilidad técnica, materiales y costos. • Resultados: Mejor comprensión de la magnitud del proyecto. • Problemas: Dificultad para crear una base de datos para la ESP32. • Soluciones: Investigar bases de datos públicas o cambiar la cámara. Julio 12: • Actividad: Reunión con mentor sobre el sistema de audio. • Resultados: Decisión de usar el DFPlayer Mini para la respuesta auditiva. • Problemas: Falta de conocimiento sobre el DFPlayer Mini. • Soluciones: Investigar su funcionamiento y compatibilidad. Julio 13 y 14: • Actividad: Investigación de la cámara ESP32, enfocándose en el reconocimiento de imágenes y su precisión en diversas condiciones de luz. Julio 17: • Actividad: Reunión con mentor. Se decidió explorar sensores ultrasónicos como alternativa a la ESP32. • Resultados: Creación de un repositorio en GitHub y búsqueda de sensores ultrasónicos. • Problemas: La ESP32 requiere mucho tiempo de calibración. • Soluciones: Evaluar sensores ultrasónicos.
